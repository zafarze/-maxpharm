# Telegram Bot интеграции 1С

Бот для врачей с функционалом финансовых уведомлений и интеграцией с системой 1С.

## Функционал
- Регистрация врачей в боте (сбор `telegram_id`).
- Вывод текущего статуса (`/status`).
- Ежемесячный планировщик (`APScheduler`) для опроса 1С и массовой рассылки уведомлений врачам.
- HTTP API (`Flask`) для ручного запуска рассылки или приема данных (вебхуки от 1С).

## Требования
- Python 3.9+
- SQLite (или любая другая БД, поддерживаемая SQLAlchemy)

## Установка и запуск

1. **Создайте виртуальное окружение:**
```bash
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate
```

2. **Установите зависимости:**
```bash
pip install -r requirements.txt
```

3. **Настройте переменные окружения:**
Скопируйте файл `.env.example` в `.env` и укажите ваши настройки (Токен бота, URL базы 1С).
```bash
cp .env.example .env
```

4. **Запустите приложение:**
```bash
python app.py
```

## Взаимодействие с 1С
В базе данных `doctors` есть поле `doctor_id` — это GUID врача в 1С.
После того как врач напишет боту `/start`, его `telegram_id` появится в базе.
Администратор должен связать `telegram_id` и `doctor_id`, чтобы бот мог запрашивать данные по этому врачу из 1С.

**Формат ответа от 1С:**
```json
{
  "status": "ok",
  "doctor_id": "GUID-12345",
  "full_name": "Иванов Иван Иванович",
  "price": 15000.00,
  "bonus": 1250.50
}
```

## API Endpoint'ы

* `POST /api/trigger_sync` — Запустить массовый опрос 1С и рассылку уведомлений (вручную).
* `POST /api/update_doctor` — Альтернативный вариант обновления данных врача (если 1С сама отправляет запросы в бота).
